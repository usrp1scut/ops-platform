Executive summary
Read first · decision lives on this page

The shell is good. The map is wrong.

The visual language — dark sidebar, light surface, dense tables, drawers, teal accent — is already calm and correct for an internal console. The problem is the top-level map: it lists backend modules in roughly the order they were built, not the work an operator does. Three changes recover most of the value.

Change 01
Promote workflow zones

Reorganise the sidebar into Operate (Connect, Sessions, Access), Inventory (CMDB, Connectivity), Govern (IAM, Audit, AWS, OIDC). Three zones map onto three distinct mental tasks.

Change 02
Split Sessions Live vs Audit

"Sessions Live" is an interactive workspace. "Audit" is a forensic table over closed sessions and recordings. They share data but not user intent — they should be siblings, not modes.

Change 03
Make permissions readable

IAM today shows users → roles. It should show capabilities: who can SSH where, who can approve, who can administer. Roles stay; they become a way of editing the matrix, not the matrix itself.

Bottom line

Keep the visual system. Re-cut the navigation around the four jobs that bring operators back to the product daily: connect, monitor, review, and govern. The rest of this deck shows what that produces.

Five problems doing most of the damage.

01

Navigation is organised by backend module, not operator job.

The sidebar reads as a list of services (CMDB, Sessions, Access, Connectivity, AWS, IAM, OIDC, Profile). An operator opening the console to do their job has to translate "I need to SSH into a host" into "Sessions > Live > SSH tab". Every flow pays this translation tax.

Sessions Live and Sessions Audit are two products in one route.

Live is a real-time, two-pane workspace (rail + terminal). Audit is a filterable, paginated forensic table. They share zero interaction patterns and yet sit behind the same tab switcher. Live feels cramped because Audit's chrome is right next to it; Audit feels buried because Live owns the URL.

Operate · the daily zone

Connect is the new front door — an asset tree + search + recent + one-click SSH/RDP. Sessions is purely live (active terminals/RDPs, my work right now). Access is request & approval.

  • Backend untouched — pure IA + routing.
  • Old URLs redirectable; nothing breaks.

Ops Platform · Design review v105 / 14
Recommendation

Pick Option A. Bank the smaller wins.

Why A

Option A keeps every existing route. The change is purely how they're grouped in the sidebar and which one acts as the front door. That means we can ship the IA shift in a single sprint, redirect legacy URLs, and measure usage before committing to anything more radical (B or C).

What "Connect" is

A new top-level surface backed by the same asset query the CMDB list runs. It defaults to the asset tree (env → vpc → host), promotes search to the header, recents at the top, and the action it offers is SSH or RDP — not "view detail". Detail still lives in CMDB.

What "Sessions" becomes

Purely the live workspace. It loses the Audit tab. Tab strip at the top of the terminal area becomes proper session tabs (multiple concurrent SSH/RDP), not a Live/Audit switcher.

What "Audit" gains

Its own route under Govern. Filters get more room. Recording inspector becomes a full page, not a row action. Cross-links into IAM (who used this grant?) and Access (was this request approved?).

Tradeoffs we are accepting
  • One more top-level item in the sidebar (Audit) — softened by grouping.
  • "Connect" is similar enough to CMDB Tree view that some users will need a beat to learn the difference. We address this with a one‑liner on the empty state and a "view in CMDB" link on every detail.
  • Existing operators' muscle memory ("Sessions → Audit tab") breaks. We mitigate with a sticky deprecation note for 30 days and an automatic redirect.
What this unlocks next
  • IAM redesign — capability matrix slots cleanly into the Govern group.
  • Overview page becomes a real action surface (active sessions, pending approvals, recent recordings) once Audit moves out.
  • Telemetry on the Operate group tells us whether we should later collapse to Option B.

Decision

Adopt Option A in the next iteration. Re-evaluate against Option B after one quarter of usage data on the Operate group.

Ops Platform · Design review v113 / 14
Do not change

What's already right.

This list matters as much as the redesign. Most of the visual and interaction language is correct for an internal operations console; this proposal does not touch it.

Visual language
Dark sidebar + light surface

Linear/Notion-flavoured. Sidebar reads as chrome, content reads as work. Teal accent. Inter + JetBrains Mono. All tokens (--color-accent #0d9488, --color-fg #18181b, --space-3 12px) stay as defined in tokens.css.

Density
Compact, 13px base

The product earns its small text. Operators scan 99-row tables and 20-host VPC groups — generous whitespace would be a regression. --text-base 13px and --space-3 12px stay.

Patterns
Toolbar + table + drawer

The drawer-for-detail / modal-for-create / table-as-primary pattern is correct for inventory work. CMDB keeps it. Audit inherits it. Only Sessions Live and Connect break from it, because they are workspaces, not lists.

Theme
Light/dark toggle

The token system already covers both modes. Nothing in this proposal forces dark or light — every new surface uses the same variables and inherits both themes for free.

Approval model
Mine / Approve in Access

The two-mode toggle inside Access (Mine vs Approve) is appropriate — same surface, two filters, same actions. We don't recommend changing it.

Terminal/RDP
In-browser session

The decision to render terminals inline (not pop-out a native client) is the right operator-console move — recording, idle limits, and audit only work because the platform owns the pipe.

Copy & tone
Quiet, factual, no marketing

"No active sessions · Pick an asset from the left rail to open a new session." That voice — short, declarative, no exclamation marks — is correct. Keep it.

Asset rail metaphor
env → vpc → host grouping

The hierarchy currently used in Sessions Live's left rail is exactly the right grouping for asset discovery. We're not changing it — we're moving it to a more discoverable surface (Connect).

Legacy escape
"Old portal" link

Keep the "Old portal" affordance for one release after IA rollout. Internal users have muscle memory; a deprecation ramp is cheap insurance.

Net change

Zero changes to colour, type, density, theme, drawer pattern, table pattern, terminal chrome, or copy tone. The IA shifts; the surfaces it produces use the same building blocks the team has already invested in.
